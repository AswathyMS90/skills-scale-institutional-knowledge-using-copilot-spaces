# OctoAcme Project Management Docs

This README provides a short summary of OctoAcme's project management processes and direct links to the full process documents in the docs/ folder.

## Summary

OctoAcme follows an iterative, customer-first delivery model with clear ownership and lightweight artifacts. Key practices include:

- **Defined roles**: Project Manager (PM), Product Manager (PdM), Developers, and QA
- **Project initiation** with a One-pager to define problem, goals, and success metrics
- **Planning** with prioritized backlog, estimates, Definition of Done, and risk register
- **Execution** using a project board (Backlog → Ready → In Progress → In Review → QA → Done), small PRs, CI checks, and regular demos
- **Release & deployment** checklists, smoke tests, and rollback plans
- **Regular retrospectives** and action-item tracking to continuously improve

## Core Principles

- **Customer-first**: Prioritize customer value and usability
- **Iterative delivery**: Deliver small, testable increments
- **Clear ownership**: Each project has named PM and Product Lead
- **Data-informed decisions**: Measure impact and iterate based on evidence
- **Psychological safety**: Encourage feedback and learning

## How to Use These Docs

- **New to OctoAcme?** Start with the [Project Management Overview](octoacme-project-management-overview.md) for a quick orientation.
- **Proposing a new project?** Use the One-pager template in [Project Initiation Guide](octoacme-project-initiation.md).
- **Running a sprint or release?** Follow the [Planning](octoacme-project-planning.md) and [Execution & Tracking](octoacme-execution-and-tracking.md) docs.
- **Need to escalate a risk or communicate status?** Refer to [Risks & Communication](octoacme-risks-and-communication.md).
- **Preparing for a release?** Check the [Release & Deployment](octoacme-release-and-deployment.md) guide.
- **Capturing learnings?** Use the [Retrospective & Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md) process.
